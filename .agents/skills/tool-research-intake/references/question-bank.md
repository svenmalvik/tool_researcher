# Question Bank

Use these questions to create a usable research brief before broader analysis starts.

## Mandatory Questions

- What tool are we evaluating?
- What problem or job should this tool solve?
- Who will use it, approve it, or be affected by it?
- About how many people may use this tool?
- What kinds of users are they: PMs, developers, designers, operators, executives, or other roles?
- What similar tools are already in use today?
- Which teams or workflows use those current tools?
- What is working well with the current tools?
- What is missing, slow, expensive, or risky with the current tools?
- What capabilities are mandatory?
- What constraints matter most: budget, security, compliance, integrations, timeline, geography, or vendor preferences?
- What decision are we trying to make: invest, pilot, replace, expand, or reject?
- When does the decision need to be made?
- Do you already have any notes, documents, links, prior research, vendor material, or internal context we should use?

## Baseline Questions

Ask these when current tools already exist:

- Are the current tools official standards or ad hoc choices?
- Are there contracts, sunk costs, or migration barriers tied to them?
- Are there known user complaints or adoption issues?
- Roughly how many people use the current tools today?
- Do different user roles need different licenses or plan tiers?
- Which parts of the workflow are already handled well enough today?
- What would have to be materially better to justify change?

## Optional Depth Questions

Use these when the evaluation needs more precision:

- Which integrations are non-negotiable?
- Which metrics define success after adoption?
- Is this a team-level decision or an organization-wide platform decision?
- Is there a budget range, target cost per user, or spending ceiling?
- Does the user need a recommendation, a shortlist, or a full comparison package?
- Are there specific competitors that must be included even if they are not currently in use?
- Are there internal assumptions, political constraints, or prior decisions that could affect the recommendation?

## Intake Output Format

Summarize the answers into a brief that includes:

- Scope of the evaluation
- Expected user count and user-role assumptions
- Current-state baseline candidates
- Decision criteria
- User-provided context and supporting material
- Known unknowns
