# Tool Research Intake

## Overview

Use this skill to scope the research before the project moves into market analysis. The output of this skill is a research brief that is specific enough to guide the baseline and analysis work, including pricing assumptions based on expected users.

## Workflow

1. Confirm the candidate tool and the decision the user is trying to make.
2. Ask the required intake questions from `references/question-bank.md`.
3. Explicitly ask what similar tools are already in use today.
4. Capture what is working well and poorly with the current tools.
5. Ask how many people may use the tool and what kinds of users they are.
6. Ask whether the user has any additional information, documents, links, or prior analysis that should shape the research.
7. Create an initial task list for the full workflow.
8. Turn the answers into a concise research brief with clear open questions.

## Required Output

Produce a research brief with these sections:

- Candidate tool
- Problem to solve
- Users and stakeholders
- Expected user count and user roles
- Current tools in use
- Pain points and desired outcomes
- Constraints and non-negotiables
- Decision criteria
- User-provided context and source material
- Open questions
- Initial task list

## Handoff

If current tools are already in use, hand the brief to `tool-research-baseline` next.

If there is no current tool baseline, state that the evaluation is effectively greenfield and hand the brief to `tool-research-analysis`.
