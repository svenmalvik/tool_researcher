# Tool Research Baseline

## Overview

Use this skill to study the tools already in use before evaluating the candidate tool. This prevents the analysis from treating the decision as if it starts from zero.

## Workflow

1. Read the intake brief and list the current tools in scope.
2. Research each current tool from the lenses in `references/baseline-comparison-framework.md`.
3. Identify what the current stack already does well.
4. Identify pain points, switching costs, and risk created by the current state.
5. Produce a baseline summary and the comparison criteria the candidate tool must meet or exceed.
6. Update the task list to mark the baseline work complete.

## Required Output

Produce the following:

- A current-state summary
- A baseline comparison table across the incumbent tools
- A short list of strengths worth preserving
- A short list of weaknesses or gaps that justify change
- A comparison rubric for the candidate tool analysis

## Handoff

Pass the baseline summary and rubric to `tool-research-analysis`.

If the user has no meaningful incumbent tools, state that clearly and skip this skill.
